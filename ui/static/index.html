AI fit scoring — how the 0–100 score is actually set
The score you see is set by an AI model (Claude Haiku) that reads each job against your CV + LinkedIn summary and follows the rubric below — judged semantically, not by keyword matching. It works in two steps:
  1. Skills & qualifications match sets the ceiling — a weak match caps the score low no matter how attractive the role; a strong, specific overlap with your CV pushes it high.
  2. Role-type tier places it within that ceiling — remote-anywhere 0→1 founder-led (top) → AI-for-vertical scale-ups → remote Europe HoP → UK-hybrid IC (lower).
Geo & language gates are applied separately (Filters). The keyword weights below are only a fallback used when no AI key is configured (and they power the “skills considered ✓” chips).
Weighted factors — the 2nd-scale "weighted" score (also the no-AI fallback)
This is the weighted score shown beneath each AI score in the list. Each factor contributes weight × how many of its keywords appear in the job; weights must sum to 100. Full remote is mode-based (no keywords). Saving re-scores the list.
Board search queries — what query-based boards (e.g. Adzuna) search for
One query per line. The default applies to every query-based board; add a per-board override to search differently there.
Learned preferences — your manual CV edits that teach future drafts
Captured each time you edit a highlighted change in a draft (with a rationale). Injected into the model for every new draft. Prune anything you don't want it to learn.
