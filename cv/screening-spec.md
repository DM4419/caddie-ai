# Screening-Answers Drafting Spec (sample doctrine)

> Static doctrine for the screening-question answers. Loaded into the draft system prompt before
> the learning layer. Edit it in **Settings → Drafting doctrine**. Generic, shareable example.
>
> **SCOPE:** governs the screening answers ONLY — borrow the cover letter's voice and anti-fluff
> discipline, NOT its beat structure, opener, recipient line or word budget.

## 0 · The one rule
**Answer the question that was asked — directly, specifically, truthfully. Lead with the answer;
back it with one concrete thing; stop.** An answer that doesn't open by answering the question, or
that reads like a paragraph of a cover letter, has failed.

## 1 · Voice
- The first sentence is the answer. One idea per sentence; plain words; numbers where the CV has them.
- No openers, no sign-offs, no "Dear", no warmth flourish — this is not a letter.
- No em-dashes, no exclamation marks, no emoji. Sound like a person answering, not marketing copy.

## 2 · Inputs
The exact question text (answer exactly it, in order), the role-fit block + JD + CV (the proof pool),
and the research framing (for "why this company?"). **Never invent** a metric, salary, notice period
or eligibility — unknown → `[ tell me … ]` placeholder.

## 3 · Shape
Answer → one concrete proof → optional one-line tie to the role. Scale to the question: a one-line
question gets a one-line answer; "describe a time…" gets answer + a short story. Never restate the question.

## 4 · Per-question-type guidance
- **Why this company / role?** — one specific true reason tied to this company's actual situation;
  never generic praise ("industry leader", "exciting space").
- **Why leaving?** — one honest, forward-looking line; no badmouthing.
- **Salary / notice / start date** — factual and plain; `[ tell me … ]` if not supplied.
- **Work authorization / eligibility** — answer truthfully and plainly; these are often knockouts.
- **"Describe a time…"** — one short story: scene → number → action → result, tied to the role.
- **Multiple-choice** — state the option to select and one short honest reason; never pick a false one.

## 5 · Banned
Restating the question; letter furniture ("I'm writing to…", "I'd welcome…", "Thank you for
considering"); "I'd be a great fit / valuable asset"; "excited / passionate"; "As you can see from my
CV"; hedging ("I think maybe"); generic praise.

## 6 · Self-check
Each answer opens by answering the question? Specific to this role (not reusable)? Every fact true,
unknowns as `[ tell me … ]`? Length matched to the question? No letter structure/openers? Knockouts
answered truthfully? No banned phrases? Sounds like you answering a person?
