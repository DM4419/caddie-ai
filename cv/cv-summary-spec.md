# CV Summary Drafting Spec (sample doctrine)

> Static doctrine for the CV **Summary** (the top-of-CV positioning block). Loaded into the draft
> system prompt before the learning layer. Edit it in **Settings → Drafting doctrine**. Generic,
> shareable example — replace with your own.

## 0 · The one rule
**Compress proof, not personality. No line without a number or a verifiable specific.**
A summary is scanned in seconds and parsed by ATS first; every line carries evidence or sets up the
line that does. Pure self-characterisation must be anchored to something concrete or cut.

## 1 · Voice
- Direct and sparse, one idea per sentence. State the number plainly; let it do the work.
- One line of character allowed (the signature line) and nowhere else.
- Active voice, concrete nouns. First-person or implied, consistent. No em-dashes, no exclamation marks.

## 2 · Inputs
Research `emphasis` (decides which proof number leads per role), the role-fit block (the number
pool), JD keywords (mirror the posting's terms), and the base CV (the only source of metrics).
**Never invent or round up** a number — unknown → `[ tell me … ]` placeholder.

## 3 · Structure (4 lines)
| Line | Job |
|---|---|
| **Position** | Identity + strongest differentiator. No years, no target title. |
| **Proof** | 1–2 quantified milestones that back the Position. ≥1 number, true to the CV. |
| **Range + through-line** | The verticals (ATS keywords) bound by one explicit connective thesis. |
| **Signature** | One human line of operating point of view. The only character line. |

## 4 · Per-line guidance
- **Position** — specific enough that it couldn't headline a generic product CV; fold operating
  style in as a clause, don't give it its own line.
- **Proof** — at least one real number, inside a clause (not a bare stat or a stat-stuffed list).
- **Range** — state the through-line explicitly ("the through-line is the same: …"), so multiple
  verticals read as deliberate range, not drift.
- **Signature** — short, earned, in your own voice. No aspirational filler.

## 5 · ATS guardrail
Put keywords inside achievement sentences, not lists. Mirror the JD's exact terminology. Never
overstate a title or metric.

## 6 · Banned phrases
"results-driven / seasoned / dynamic / passionate", "proven track record", "high-impact /
high-velocity", "cross-functional squads" (use "small teams"), "world-class / best-in-class",
"committed to delivering", "spearheaded / synergy / leverage" as filler.

## 7 · Self-check
≥1 quantified milestone true to the CV? No line is pure self-description without an anchor?
Through-line stated explicitly? No years/target-title in line 1? No banned filler? Keywords in
sentences, terminology mirrored? 3–5 lines, ~50–80 words, no em-dashes? Sounds like a person?
